1、搭建备份服务器（backup服务器之rsync服务）
工具：Rsync
能实现本地和远程的服务器

rsync 保持属性复制
rsync vzrtopg /etc/hosts /mnt/
同步功能，删除，和本地同步
rsync -avz –delete /tmp1/ /mnt/

普通复制，同等于CP
rsync -vzrtopg /etc/host/ /tmp 复制（带属性）
rsync -avz –delete /etc/host/ /tmp 同步删除（以/tc/host）为原
远程复制，复制文件到远端，通过隧道
rsync -avz /etc/host -e ‘ssh -p 22’ oldboy@10.0.0.31:/tmp
-e使用通信协议，隧道
-p保持权限
rsync -avz /etc/host -e ‘ssh -p 22’ oldboy@10.0.0.31:/tmp ssh key 免密钥，一班用于定时任务
服务端配置文件
/etc/rsyncd.conf,默认是不存在，需要创建

#rsync_config__________________start
#created by oldboy 15:01 2007-6-5
#QQ
###rsyncd.conf start##
uid = rsync
gid = rsync
use chroot = no
max connections = 200
timeout = 300
pid file = /var/run/rsyncd.pid
lock file = /var/run/rsync.lock
log file = /var/log/rsyncd.log
[backup]
path = /backup
ignore errors
read only = false
list = false
hosts allow = 172.16.1.0/24 #允许连接的IP，一般是内网
hosts deny = 0.0.0.0/32 #拒绝连接的IP
auth users = rsync_backup
secrets file = /etc/rsync.password
#rsync_config__________end
########################################

添加用户
useradd rsync -s /sbin/nologin -M 虚拟用户
启动服务
rsync –daemon
查看是否启动
ps -ef|grep rsync|grep -v grep
创建目录
mkdir /backup
改变backup权限
chown rsync.rsync /backup/
查看文件有没有
ll /etc/rsync.password
如果没有泽创建文件
touch /etc/rsync.password
编辑文件
rsync_backup:oldboy#用户名：密码
然后更改权限
chmod 600 /etc/rsync.password
查看端口
lsof -i 873
netstat -lntup|grep 873

3.熟练部署过程（原理）
tail /var/log/rsyncd.log
命令示例1：
rsync -avz /backup/ rsync_backup@172.16.1.41::backup/ –pasword-file=/etc/rsync.password
命令示例2:
rsync -avz /backup rsync://rsync_backup@172.16.1.41/backup/ –password-file=/etc/rsync.password

拷贝服务器端模块内容(从客户端拉)
rsync -avz rsync_backup@172.16.1.41::backup/ /backup/ –password-file=/etc/rsync.password
排除同步
排除单个文件：
rsync -avz –exclude=a /backup/ rsync_backup@172.16.1.41::backup –passwork-file=/etc/rsync.password
排除两个
rsync -avz –exclude={a,b} /backup/ rsync_backup@172.16.1.41::backup –password-file=/etc/rsync.password
文件排除方法
rsync -avz –exclude-from=paichu.log /backup/ rsync_backup@172.16.1.41::backup –password-file=/etc/rsync.password
无差异同步
rsync -avz –delete /backup/ rsync_backup@172.16.1.41::backup –password-file=/etc/rsync.password
限速
rsync -avz –bwlimit=10 /backup rsync_backup@172.16.1.41::backup –password-file=/etc/rsync.password

dd if=/dev/zero of=test1 bs=1M count=128 创建128个1M的文件

rsync -rlptoDvz /etc/host* rsync_backup@172.16.1.41::backup/
rsync3.1.2版本参数用-avz会出现问题，需要更换成 -rlptoDvz，和原来参数对比缺少一个-g参数
